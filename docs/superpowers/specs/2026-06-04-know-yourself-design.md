# Know Yourself - AI 自我认知应用设计文档

## 项目定位

一个 AI 自我认知应用，通过持续对话帮助用户深入了解自己。核心差异化在于**洞察力**：不只是聊天，而是能发现用户自己没意识到的行为模式、情绪规律和内在矛盾。

- 目标用户：先自用，后产品化
- 核心能力：洞察（发现用户未意识到的模式与矛盾）
- 使用场景：日常随聊、遇到问题时求助、定期深度探索
- 对话风格：根据场景自适应（温暖/犀利/苏格拉底式/朋友感）

## 技术选型

- 前端：Next.js 14+ (App Router)
- UI 组件：shadcn/ui
- 状态管理：zustand
- 图表：Recharts
- 数据库：Supabase（PostgreSQL + Auth + Realtime + pgvector）
- AI 接口：DeepSeek（V4 Pro + V4 Flash），抽象层支持切换
- 部署：Vercel（前期）

## 对话系统设计

### 对话流程

```
用户发消息
    │
    ▼
加载上下文（核心画像摘要 + 最近 5 轮对话 + 当前情绪状态）
    │
    ▼
路由判断：这条消息需要深度思考还是轻量回应？
    │
    ├─ 轻量 → V4 Flash（日常闲聊、简短回应）
    └─ 深度 → V4 Pro（涉及情绪、决策、自我探索）
    │
    ▼
AI 生成回应（带 system prompt 指导风格自适应）
    │
    ▼
后处理：提取情绪标签、关键事件、潜在洞察线索
    │
    ▼
存储对话 + 更新短期记忆
```

### 风格自适应机制

| 检测到的场景 | 风格策略 |
|---|---|
| 情绪低落/倾诉 | 温暖共情，先接纳再引导 |
| 探索人生方向 | 苏格拉底式提问，引导自我发现 |
| 分析具体问题/决策 | 直接犀利，列出没考虑到的角度 |
| 日常随聊 | 轻松自然，像朋友 |

场景检测由 V4 Flash 做快速分类，再传给主对话模型。

### 上下文管理

- 每次对话携带：用户核心画像（~500 tokens）+ 最近 5 轮对话（~2000 tokens）+ 相关历史片段（语义检索，~1000 tokens）
- 总上下文控制在 4000-5000 tokens，平衡效果和成本
- 语义检索用 Supabase pgvector 扩展，历史对话 embedding 后按相关性召回

## 记忆与用户画像系统

### 记忆分层

**短期记忆（当前会话）：**
- 当前对话上下文
- 本次对话情绪变化
- 临时话题标记

**中期记忆（对话摘要）：**
- 每次对话的关键信息摘要
- 提到的人物、事件、决策
- 情绪标签 + 强度
- 向量化存储，支持语义检索

**长期记忆（用户画像）：**
- 核心性格特征
- 价值观倾向
- 行为模式
- 人际关系图谱
- 长期情绪基线
- 人生主题与未解问题

### 画像数据结构

```json
{
  "personality": {
    "traits": ["内向敏感", "追求完美"],
    "values": ["自由", "真诚"],
    "confidence": 0.7
  },
  "patterns": [
    {
      "description": "面对压力时倾向于自我否定",
      "evidence_count": 5,
      "first_seen": "2026-03-01",
      "last_seen": "2026-06-01",
      "confirmed_by_user": true
    }
  ],
  "relationships": [
    { "name": "小明", "role": "同事", "dynamic": "竞争又欣赏" }
  ],
  "emotional_baseline": {
    "dominant": "平静偏焦虑",
    "triggers": ["被否定", "不确定性"]
  },
  "life_themes": [
    { "theme": "寻找职业意义", "status": "探索中" }
  ]
}
```

### 确认机制

- AI 自动积累观察，低确信度的先静默记录
- 当某个洞察积累了 3 次以上证据，或涉及核心性格判断时，在合适的时机问用户确认
- 确认方式自然嵌入对话，例如："我注意到你几次提到类似的情况，你觉得自己是不是在面对冲突时会倾向于先退让？"
- 用户可以同意、否认或补充，AI 相应更新画像

## 洞察分析引擎

### 分析流程

```
定期触发（每 5 次对话 / 每周 / 用户主动请求）
    │
    ▼
加载全量中期记忆 + 当前画像
    │
    ▼
V4 Pro 深度分析（多维度）
    │
    ├─ 纵向：同一主题跨时间的变化趋势
    ├─ 横向：不同领域间的关联和矛盾
    └─ 深层：表面行为背后的潜在动机
    │
    ▼
生成洞察结果
    │
    ├─ 更新用户画像（新模式、修正旧判断）
    ├─ 生成洞察报告（用户可查看）
    └─ 标记待确认项（下次对话时自然引入）
```

### 洞察类型

| 类型 | 示例 |
|---|---|
| 模式识别 | "你在最近三次职场冲突中都选择了沉默，但事后会反复自责" |
| 矛盾发现 | "你说重视自由，但每次做选择时都优先考虑别人的期待" |
| 变化追踪 | "相比一个月前，你谈论工作时的焦虑感明显减少了" |
| 盲点提示 | "你很少谈到自己的需求，总是从别人的角度出发" |
| 关联洞察 | "你对同事的不满和你对父亲的感受有相似的模式" |

### 触发策略

- **自动触发**：每累计 5 次对话跑一次轻量分析（V4 Flash）
- **周期触发**：每周检查，仅当本周有新对话记录时才跑深度分析（V4 Pro）；无对话则跳过
- **手动触发**：用户可以随时请求"帮我分析一下最近的状态"

### 洞察报告结构

```markdown
## 本周洞察 - 2026.06.01

### 新发现
- [模式] 描述 + 证据摘要

### 变化趋势
- [情绪] 整体走向 + 关键转折点

### 待探索
- [问题] AI 观察到但还不确定的点，邀请用户下次聊聊

### 画像更新
- 新增/修正了哪些认知
```

## 情绪追踪与可视化

### 情绪数据采集

每次对话时，V4 Flash 对每条用户消息做轻量情绪标注：

```json
{
  "timestamp": "2026-06-04T14:30:00Z",
  "emotion": "焦虑",
  "intensity": 0.7,
  "trigger": "工作deadline",
  "context_summary": "担心项目延期被领导质疑"
}
```

标注嵌入对话后处理流程，不额外调用模型，成本可忽略。

### 可视化功能

| 视图 | 作用 |
|---|---|
| 情绪时间线 | 折线图，看日/周/月的情绪波动 |
| 情绪分布 | 饼图/条形图，各情绪占比 |
| 触发因素排行 | 什么事最容易影响情绪 |
| 情绪-事件关联 | 点击某个低谷可以回溯当时的对话 |

图表技术：Recharts，数据从 Supabase 查询后按时间窗口聚合。

## 认证与多用户

- 认证方案：Supabase Auth（邮箱+密码，后续加第三方登录）
- 数据隔离：所有表通过 `user_id` 隔离，RLS 保证安全
- 多用户扩展：数据结构从一开始按多用户设计，产品化时只需加计费和用量限制

## MVP 范围

### 包含

- 对话界面（流式输出）
- 模型路由（Pro/Flash）
- 短期+中期记忆
- 用户画像自动构建
- 洞察报告生成
- 情绪标注+仪表盘

## 开发阶段

### Phase 1：基础可用（1-2 周）

- 项目搭建（Next.js + Supabase + shadcn/ui）
- 认证系统
- 基础对话界面（流式输出）
- AI 抽象层 + DeepSeek 对接
- 对话存储

### Phase 2：记忆系统（1-2 周）

- 对话摘要提取（对话结束后自动）
- 向量化存储（pgvector）
- 上下文检索（语义相似度召回）
- 用户画像数据结构 + 自动积累
- 风格自适应（场景检测 + 动态 prompt）

### Phase 3：洞察能力（1-2 周）

- 分析引擎（定期触发逻辑）
- 洞察报告生成 + 展示页面
- 画像确认交互
- 情绪标注 + 仪表盘可视化

## 技术要点

- 流式输出：Server-Sent Events（Next.js Route Handler + ReadableStream）
- 向量检索：Supabase pgvector 扩展，embedding 调用 DeepSeek 或开源模型
- AI 调用：统一抽象层，方便后续切换模型
- 前端状态管理：zustand
